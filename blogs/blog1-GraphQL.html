What is GraphQL? GraphQL Vs Rest

With REST, you generally submit your request through a specially crafted URL and send each request to a different destination-for example: / movie / 2120 or / director / 5130

With GraphQL, you can submit a declarative request for the data you are searching for in a JSON-like query, and all requests will be sent to the same destination. The schema used for the request determines the data you will return. This is a standardized, self-describing method to request the specific data required, and only the required data.

In REST, the shape and size of the resource is determined by the server. In GraphQL, the server declares what resources are available, and the client asks for what it needs at the time.

Different request types use different schemas, instead of using different endpoint URL formats, making the query mechanism more flexible. Although many REST APIs like Swagger also conform to general specifications, there is no rule that REST APT must be generated by Swagger . GraphQL provides a formal definition for the API by default.

In a sense, GraphQL is a bit like SQL (Structured Query Language). Using the data source provided by SQL, you can connect all data requests to a common end point, and the format of the request determines what kind of records you will get back. Although SQL has many different implementations, the SQL query syntax remains highly consistent between these implementations.
